# Recipes Agent Playbook

## Purpose

This repository stores Jekyll recipe posts in `_posts/*.md`.
The primary agent goal is to create or reformat recipes so they match the
existing repository style.

## Default Behavior

If the user pastes recipe text without an explicit action, treat it as a
direct request to create a new post from that recipe.

Do not ask for confirmation unless required data is truly missing.

Use the web liberally to look up recipes, techniques, or ingredient details
to help produce a better result.

## Required Workflow

1. Read 2-3 similar recent recipes from `_posts/` to match style and tone.
2. Create the post file with:
   - `./create_post.sh "<title>"`
3. Fill the generated post with complete recipe content.
4. Use existing tags as source of truth:
   - `./list_tags.sh`
5. Add appropriate tags in front matter.
6. Run a final formatting pass before finishing.

## Recipe Language And Structure

- Write recipe content in German.
- Keep Jekyll front matter as generated by `create_post.sh`.
- Keep the in-body title line as `# <Titel>`.
- A recipe may optionally have a short introductory sentence directly after
  the `# <Titel>` line (no blank line in between). This is typically a brief
  characterization, impression, or contextual note about the dish
  (e.g. "Einfach und lecker.", "Für etwa ein Ofenblech.").
  The blank line then follows after this sentence, before the first section.
- Do not add markdown subheadings (`##`, `###`, etc.).
- Plain text labels like `Teig ansetzen:` are allowed.

## Strict Formatting Rules

Follow these rules exactly:

1. Output recipe body in markdown style used by the repo.
2. Use one instruction per preparation step line.
   Follow-up sentences that qualify, constrain, or explain the same
   instruction stay on that line, e.g.:
   `Teig bestreichen. Dabei einen ca. 1 cm Rand freilassen.`
3. Use imperative/infinitive style for preparation steps.
4. Add exactly two trailing spaces at the end of each non-blank, non-ingredient
   line after the front matter (preparation steps, descriptions, section labels,
   closing remarks).
5. Do not add trailing spaces to ingredient list lines (`- ...`).
6. Keep blank lines completely empty (no spaces or tabs).
7. Unit spacing rules:
   - Use a space for spoon-style units: `1 TL`, `2 EL`.
   - Use no space for metric units: `200g`, `1kg`, `250ml`, `2l`.
   - Temperatures: `x °C` with a space before the degree sign, e.g. `180 °C`.
     Do not use `Grad` or other formats like `180°C` or `180° C`.
8. Use hyphens (`-`) for ranges, not en-dashes (`–`), e.g. `10-15 Minuten`.

## Tagging Policy

- Prefer existing tags from `./list_tags.sh`.
- Reuse established tag vocabulary whenever possible.
- Add tags that match dish type, course, or key characteristic.
- Avoid inventing new tags unless no existing tag fits.

## Title Handling

When creating a post from pasted recipe text:

1. Use the pasted markdown `# <Titel>` if present.
2. Otherwise infer a concise recipe title from the content.
3. Ask the user only if no reliable title can be inferred.

## Quality Check Before Finalizing

Before returning results, verify:

- front matter is valid and date/title are present
- tags are set and come from existing vocabulary when possible
- ingredient bullets have no trailing spaces
- all non-blank, non-ingredient lines have two trailing spaces
- blank lines are completely empty
- unit spacing follows rules (no space for metric, space for spoon-style)
- no markdown subheadings were introduced
